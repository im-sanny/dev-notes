# Operating Systems (OS) and Computer Architecture

## Role of Operating System

- **Core Function**: OS helps to run or execute programs in a system by managing hardware resources and providing an interface between hardware and software.
- **Memory Management**:
  - Program loads on **RAM** after turning on the system.
  - **RAM** provides fast, temporary storage for active programs.
  - **HDD/SSD** only store data that we need for long time (permanent storage).
- **CPU Execution**:
  - CPU starts to work at the time we start or turn on our system.
  - It keeps working until we turn the system off, executing instructions continuously.
- **Resource Management**: OS helps to manage:
  - CPU scheduling
  - Memory allocation
  - I/O operations
  - File systems
  - Security and user interface

---

## Early Computing - ENIAC Era

- **ENIAC** was the starting era general purpose computer.
- **Punch Cards Usage**: ENIAC used **punch cards** for:
  1. **Input** (data and instruction)
  2. **Output** (result)
  3. **Storage** (storing temporary data - though it also used other storage methods like vacuum tubes and delay lines)
- **Card Specifications**: ENIAC used to use **IBM's 80 column punch card** which used to have **12 rows**.
- **Manual Operation**:
  - That time there wasn't any OS so everything had to be done manually.
  - Human operators physically configured the computer by setting switches and connecting cables for each program.

---

## The Idea of Operating System

- **Origin**: The idea of operating system came from the **operator** that used to operate ENIAC with punch cards.
- **Evolution of Concept**:
  - With time, to make the computer more efficient, they automated the work of the operator with an operating system.
  - **Operator** → **Operating** (automation of human tasks)
  - **Automatic system** → **System**
- **Comparison**:
  - **Operator**: Used to help running programs in early days but **manually**.
  - **OS**: Also helps running programs but **automatically**.

---

## First Operating Systems

- **1956 - GM-NAA I/O**: First operating system (General Motors - North American Aviation for IBM 704).
- **Interface**: First OS was **terminal based not GUI** (command-line interface, no Graphical User Interface).
- **Processing Method**: Early systems used **batch processing** - operators would load batches of programs to run sequentially.

---

## Operating System Evolution

### Lineage and Influence

- **Unix** → **macOS** → **Linux** (influence and evolution path)
  - More accurately: **Unix (1969-1970)** influenced both **BSD Unix** (which led to macOS) and **Linux (1991)**.
  - **macOS** has Unix foundations (based on BSD Unix).
  - **Linux** was inspired by Unix principles.

### Input Device Timeline

- **1960** → **Keyboard** (keyboards became standard input devices).
- **1970** → **Mouse** (mouse was demonstrated in 1968, became popular in 1970s-1980s).
